Description

The website that I would like to make is an open-source messaging application akin to discord. The website will not be as ambitious due to various constraints. However, it will support the following features:

  • Create an account.
  • Sign in.
  • Sign out.
  • Create, edit and close channels.
  • Invite and remove members from a channel.
  • Send and recieve messages posted to the channel.
  • Download a log of all conversations recorded on the server.
  • Delete all conversation record upon deletion of a channel.
  • Optional Features (Features that will be implememnted if enough time):
    • Send and recieve images.
    • Send and recieve hyperlinks.
    • Send and recieve private messages.
    • Send and recieve friend requests.
    • Upload zip files to the server.
    • Email verification when creating a new account
    • Account Recovery


The goal of the project is to allow a user to create channels and converse with other members of the channel. The emphasis will be placed on using the website for teamwork and work related interactions. The project has several constraints that will impact the size of the project. Namely, I also have two heavier projects in Comp 345 and Comp 371. Consequently, I need to ensure that I assign myself a workload that I can manage given this circumstance. It is also important to note that this project is done on individually. It is not meant to be used as a gaming social media platform, but a serious privacy and security focused professional tool. In otherwords, the purpose of the website is to provide a free and open-source alternative to Facebook's messenger, Discord or Twitter for those who want to get work done.

Create Channel

Creating a channel on the website will be a similar process to creating a channel on discord. Unlike discord, this website will not support voice communication. Therefore, there will be no Voice Channels. Furthermore, discord works by using servers that are composed of one or multiple channels. Here this is not the case. As such the layout will be very simple. A rectangular box where the user inputs the name of their channel and whether it is public or private. A public server can be joined by anyone without needing an invitation.
